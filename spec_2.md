# Проект: TempMonitor v2 — Полная спецификация

## Архитектура

- **Плата:** ESP32-WROOM-32 / DevKitC
- **Фреймворк:** ESP-IDF 5.x + PlatformIO
- **Датчики:** 5× MAX31855 (термопары K-типа)
- **Дисплей:** 1.8" TFT ST7735S, 128×160 px, SPI
- **Управление:** 4 физические кнопки (INPUT_PULLUP)
- **Питание:** 5 В DC вход
- **Связь с ПК:** Serial JSON 115 200 baud → USB

|-----------|-----------|-------------------|---------|
| MOSI      | 23        | SDA / MOSI        | 3.3 В   |
| CLK       | 18        | SCK / CLK         | 3.3 В   |
| CS        | 5         | CS / CE           | 3.3 В   |
| DC        | 22        | DC / RS / A0      | 3.3 В   |
| RST       | 4         | RES / RST         | 3.3 В   |
| Подсветка | 21        | BL / LED          | 3.3 В (PWM, LEDC ch.7) |

> MISO дисплея не подключается (ST7735S write-only в данной конфигурации).

### ESP32-WROOM-32 → 5× MAX31855 (программный SPI, общая шина)

| Сигнал      | ESP32 GPIO | Назначение                    |
|-------------|-----------|-------------------------------|
| Sensor CLK  | 14        | Тактовый сигнал (bit-bang)    |
| Sensor MISO | 35        | Данные от датчика (input-only)|
| CS — T1     | 25        | Chip Select, MAX31855 #1      |
| CS — T2     | 26        | Chip Select, MAX31855 #2      |
| CS — T3     | 27        | Chip Select, MAX31855 #3      |
| CS — T4     | 32        | Chip Select, MAX31855 #4      |
| CS — T5     | 33        | Chip Select, MAX31855 #5      |

Каждый MAX31855: VCC → 3.3 В, GND → GND, MISO → GPIO 35, CLK → GPIO 14, CS → свой пин.

> GPIO 35 — input-only на ESP32, подтяжка задаётся программно (GPIO_PULLUP_ENABLE).  
> GPIO 6–11 — зарезервированы под flash, не использовать.

### ESP32-WROOM-32 → Кнопки

| Кнопка | GPIO | Действие                          | Подключение             |
|--------|------|-----------------------------------|-------------------------|
| BTN0   | 0    | Следующая страница (Grid → Stats) | Кнопка на GND, INPUT_PULLUP |
| BTN1   | 13   | Переключить °C / °F              | Кнопка на GND, INPUT_PULLUP |
| BTN2   | 16   | Яркость 100→50→15→100%           | Кнопка на GND, INPUT_PULLUP |
| BTN3   | 17   | Сброс Min / Max                  | Кнопка на GND, INPUT_PULLUP |

> BTN0 = GPIO 0 — это также кнопка BOOT на DevKitC. В нормальной работе используется как пользовательская. При нажатии во время загрузки переводит в режим прошивки — это штатное поведение.

## Схема питания (вход 5 В)

```
БП 5V DC
   │
   ├─[Ferrite bead L1]─┬─ 5V rail → VIN ESP32 DevKitC
   │                   │           (AMS1117-3.3 onboard → 3.3V)
   │                   ├─ C1: 470 мкФ электролит (bulk decoupling)
   │                   └─ C2: 100 нФ керамика (HF decoupling)
   │
   └─ GND
```

Если **не используется DevKitC** (чистый модуль ESP32-WROOM-32):
```
5V → LDO AMS1117-3.3 (или XC6206, HT7333) → 3.3V rail
           │
           ├─ C_in:  10 мкФ + 100 нФ (вход LDO)
           └─ C_out: 10 мкФ + 100 нФ (выход LDO)
```

Локальная развязка на каждом MAX31855:
```
3.3V ─── 100 нФ (0402, макс. близко к VCC пину) ─── GND
```

## Защита от EMI индукционного нагревателя

### Почему возникает шум

Индукционный нагреватель генерирует мощное переменное магнитное поле (обычно 20–100 кГц). Это поле наводит ЭДС в петлях, которые образуют провода термопар, SPI-линии и дорожки платы. MAX31855 имеет чувствительный АЦП — даже десятки милливольт наводки дают ошибку в несколько градусов.

### Аппаратные меры (в порядке приоритета)

#### 1. Скрутка и экранирование проводов термопары ← самое важное
- Использовать **экранированный кабель термопары** (тип K, экранированный, например PTFE-изоляция).
- Экран заземлить **только с одной стороны** — со стороны MAX31855 (на GND платы). С другой стороны экран оставить незаземлённым, иначе получится токовая петля.
- Если экранированного нет — минимум **скрутить два провода** термопары между собой: скрутка сокращает площадь петли и снижает наводку.

#### 2. RC-фильтр на линии MISO
```
MAX31855 DO ──[R: 33 Ω]──┬── GPIO 35 ESP32
                          │
                        [C: 100 пФ]
                          │
                         GND
```
Постоянная времени: τ = 33 × 100e-12 = 3.3 нс.  
Срезает частоты выше ~50 МГц, не влияет на SPI до 10 МГц.

#### 3. Конденсаторы на входах термопары MAX31855
Между T+ и T- (прямо у ножек MAX31855):
```
T+ ──┬──────────── (к термопаре)
     │
   [C: 10 нФ]  ← режет дифференциальную наводку
     │
T- ──┴──────────── (к термопаре)
```
Дополнительно каждый вывод на GND через 1 нФ:
```
T+ ─[1 нФ]─ GND   (режет синфазную наводку)
T- ─[1 нФ]─ GND
```

#### 4. Ферритовые кольца / бусины
- Надеть ферритовое кольцо (например Fair-Rite 2643625002 или аналог) на **оба провода термопары** вместе, как можно ближе к плате.
- На кабели питания 5В вблизи нагревателя.
- На USB-кабель, если он рядом с зоной нагрева.

#### 5. Экранирование платы
- Поместить плату в **металлический корпус** (алюминий или сталь), заземлённый на GND схемы.
- Крышка корпуса снижает влияние электрического поля (E-field), ферритовые кольца — магнитного (H-field).

#### 6. Физическое расстояние
- Минимум **20–30 см** между индукционной катушкой и платой.
- Провода термопар прокладывать **перпендикулярно** силовым проводам нагревателя (не параллельно), если приходится идти рядом.

## Рекомендации по разводке платы (PCB Layout)

### Слои
Рекомендуется **двухслойная** PCB:
- **Top:** компоненты + сигнальные дорожки
- **Bottom:** сплошной полигон GND (ground plane)

Ground plane убирает большинство проблем с EMI и снижает импеданс GND.

### Зонирование (разделение аналога и цифры)

```
┌─────────────────────────────────────────────┐
│  Цифровая зона                              │
│  ESP32 | ST7735S | Кнопки | USB             │
│                                             │
├── единая точка соединения GND (single point)│
│                                             │
│  Аналоговая зона                            │
│  MAX31855 × 5 | Разъёмы термопар            │
│  RC-фильтры на MISO | C на T+/T-            │
└─────────────────────────────────────────────┘
```

Токи цифровой земли (переключение GPIO, SPI) не должны протекать через аналоговую зону.

### Развязывающие конденсаторы

| Компонент     | C рядом с VCC    | Монтаж              |
|---------------|-----------------|---------------------|
| ESP32 модуль  | 10 мкФ + 100 нФ | Макс. близко к VIN  |
| Каждый MAX31855 | 100 нФ (0402) | Вплотную к VCC пину |
| LDO (если есть) | 10 мкФ + 100 нФ | Вход и выход        |
| ST7735S       | 100 нФ          | Рядом с VCC         |

### Дорожки

| Сигнал              | Ширина дорожки | Примечание                         |
|---------------------|---------------|------------------------------------|
| GND / Power 5V      | ≥ 1.5 мм      | Или полигон                        |
| Power 3.3V          | ≥ 0.8 мм      |                                    |
| SPI (CLK, MOSI, CS) | 0.2–0.3 мм    | Держать короткими, рядом с GND     |
| Sensor MISO (GPIO35)| 0.2 мм        | RC-фильтр у MAX31855, не у ESP32   |
| Thermocouple T+/T-  | 0.3 мм        | Вести параллельно, минимум петли   |

### Разъёмы термопар
- Использовать **разъёмы типа K** (с компенсацией холодного спая), или просто клеммники — тогда холодный спай внутри MAX31855 (он это умеет).
- Разместить разъёмы **на краю платы**, чтобы провода не шли поверх цифровых цепей.
- RC-фильтры (T+/T- → GND) ставить **непосредственно у разъёма**, до MAX31855.

### Пример расположения компонентов

```
[Разъём питания 5V]──[Ferrite]──[C bulk]──[LDO 3.3V]──[ESP32]──[ST7735S]
                                                           │
                                          [MAX31855 #1]──[Разъём T1]
                                          [MAX31855 #2]──[Разъём T2]
                                          [MAX31855 #3]──[Разъём T3]
                                          [MAX31855 #4]──[Разъём T4]
                                          [MAX31855 #5]──[Разъём T5]
                                              ↑
                                    RC-фильтры здесь, у каждого
```

## Программное обеспечение (краткая сводка)

### platformio.ini (env:monitor)
```ini
[env:monitor]
platform           = espressif32
board              = esp32dev
framework          = espidf
board_upload.speed = 921600
monitor_speed      = 115200
build_flags        = -DCORE_DEBUG_LEVEL=3
board_build.cmake_extra_args = -DNODE_TARGET=monitor "-DSDKCONFIG_DEFAULTS=sdkconfig.monitor"
```

### sdkconfig.monitor
```
CONFIG_ESPTOOLPY_FLASHSIZE_4MB=y
CONFIG_ESPTOOLPY_FLASHSIZE="4MB"
CONFIG_ESPTOOLPY_FLASHMODE_DIO=y
CONFIG_ESPTOOLPY_FLASHFREQ_80M=y
```

### Фильтрация шума (в коде)
```
raw → circular_buffer[5 выборок] → median(5) → EMA(α=0.3) → filtered_temp
```
- Медиана срезает выбросы (1–2 аномальных значения из 5 = 1 секунда).
- EMA сглаживает оставшийся дрейф.
- При ошибке датчика (NAN): фильтр не обновляется, экран показывает ERR.

### Дисплей (страницы)
| Страница | Содержимое                              | Переключение |
|----------|-----------------------------------------|--------------|
| 0 — Grid | 5 датчиков сеткой 3+2, текущие значения| BTN0         |
| 1 — Stats| Таблица Now / Min / Max для T1–T5      | BTN0         |

### Serial JSON (200 мс)
```json
{"timestamp": 12345, "top": [T1, T2, T3, T4], "bottom": [T5], "status": "OK"}
```
