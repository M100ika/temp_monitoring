# Проект: Распределенная система мониторинга температуры (5 точек)

## Архитектура системы
- **Топология:** Звезда (ESP-NOW).
- **Node_Top (Верх):** 1x ESP32-C6, 4x MAX31855, LCD 1.47", 4 кнопки.
- **Node_Bottom (Низ):** 1x ESP32-C6, 1x MAX31855, LCD 1.47", 4 кнопки.
- **Gateway (ПК):** 1x ESP32-C6 (приемник), подключение к ПК по USB-Serial.

## Технические параметры
- **Частота опроса:** 5 Гц (200 мс на цикл).
- **Питание:** Общая линия 5V DC (БП Mean Well HDR-15-5 + фильтр DL-20T1).
- **Связь:** ESP-NOW (беспроводная, 2.4 ГГц).
- **Интерфейс датчиков:** Программный/Аппаратный SPI для MAX31855.

## Распределение GPIO (ESP32-C6)
### Общее для Node_Top и Node_Bottom:
- **LCD:** MOSI(6), SCLK(7), CS(14), DC(15), RST(21), BL(22). ← Waveshare official pinout
- **Кнопки:** GPIO 0, 1, 2, 3 (с подтяжкой INPUT_PULLUP).

### Датчики (Node_Top):
- **SoftSPI:** CLK=GPIO5, MISO=GPIO9 (shared).
- **SPI CS:** GPIO 18, 19, 20, 23 (GPIO21 занят LCD RST).

### Датчики (Node_Bottom):
- **SPI CS:** GPIO 18.

## Программное обеспечение ПК (Desktop App)
- **Назначение:** Визуализация данных в реальном времени и логирование.
- **Стек:** Python 3.12, PySide6 (Qt), pyqtgraph.
- **Интерфейс:**
    - **Графики:** 5 независимых линий (по одной на датчик).
    - **Таблица:** Текущие значения (T1-T4 Node_Top, T5 Node_Bottom) + статус связи.
    - **Управление:** Кнопка выбора COM-порта, кнопка "Старт/Стоп записи".
- **Логирование:** Сохранение данных в формате `.csv` (Timestamp, T1, T2, T3, T4, T5).
- **Дистрибуция:** Сборка в один исполняемый файл (.exe) через PyInstaller.

## Протокол передачи данных (Serial JSON)
Gateway должен выдавать в Serial порт данные следующего вида каждые 200мс:
`{"timestamp": 12345, "top": [25.1, 25.2, 25.4, 25.0], "bottom": [26.1], "status": "OK"}`
